# Django migrations — `my-api-project`

## Config

- `manage.py` — entry point
- `apps/<app>/migrations/` — one directory per app
- `apps/<app>/migrations/0001_initial.py` — initial migration

## Procedure

```bash
# 1. Generate migration (autogenerate is a starting point, not final)
python manage.py makemigrations <app_name>

# 2. Read the generated file in apps/<app>/migrations/
# 3. Edit if needed — see templates below

# 4. Check for pending migrations
python manage.py migrate --check

# 5. Apply locally
python manage.py migrate

# 6. Verify the migration is reversible (test in dev)
python manage.py migrate <app_name> <previous_number>
python manage.py migrate <app_name>

# 7. Add a test that exercises the new column
pytest -q -m database
```

## Templates

### Add nullable column

```python
# Generated by makemigrations — usually correct as-is
from django.db import migrations, models


class Migration(migrations.Migration):
    dependencies = [
        ("<app>", "0001_initial"),
    ]

    operations = [
        migrations.AddField(
            model_name="resource",
            name="priority",
            field=models.IntegerField(null=True, blank=True),
        ),
    ]
```

### Add NOT NULL column with default (safe on populated tables)

```python
operations = [
    migrations.AddField(
        model_name="resource",
        name="status",
        field=models.CharField(max_length=32, default="draft"),
    ),
]
```

### Add NOT NULL column without default (use a data migration first)

```python
# Migration A: add nullable, backfill
# apps/<app>/migrations/0003_add_score_nullable.py
from django.db import migrations, models


def backfill_score(apps, schema_editor):
    Resource = apps.get_model("<app>", "Resource")
    Resource.objects.filter(score__isnull=True).update(score=0)


class Migration(migrations.Migration):
    dependencies = [("<app>", "0002_...")]

    operations = [
        migrations.AddField(
            model_name="resource",
            name="score",
            field=models.IntegerField(null=True, blank=True),
        ),
        migrations.RunPython(backfill_score, migrations.RunPython.noop),
    ]
```

```python
# Migration B: set NOT NULL once backfilled
# apps/<app>/migrations/0004_score_not_null.py
from django.db import migrations, models


class Migration(migrations.Migration):
    dependencies = [("<app>", "0003_add_score_nullable")]

    operations = [
        migrations.AlterField(
            model_name="resource",
            name="score",
            field=models.IntegerField(),
        ),
    ]
```

### Add FK + index

```python
operations = [
    migrations.AddField(
        model_name="answer",
        name="idea",
        field=models.ForeignKey(
            to="ideas.idea",
            on_delete=models.CASCADE,
            related_name="answers",
            db_index=True,
        ),
    ),
]
```

### Composite index

```python
operations = [
    migrations.AddIndex(
        model_name="resource",
        index=models.Index(
            fields=["created_by", "is_deleted"],
            name="ix_resources_user_active",
        ),
    ),
]
```

### Data migration (separate from schema)

```python
# apps/<app>/migrations/0005_backfill_display_name.py
from django.db import migrations


def set_display_name(apps, schema_editor):
    Resource = apps.get_model("<app>", "Resource")
    for resource in Resource.objects.filter(display_name__isnull=True):
        resource.display_name = resource.name
        resource.save(update_fields=["display_name"])


class Migration(migrations.Migration):
    dependencies = [("<app>", "0004_...")]

    operations = [
        migrations.RunPython(set_display_name, migrations.RunPython.noop),
    ]
```

## Hard rules

1. **Always reversible**: every migration must be reversible. If `RunPython` is used, provide a reverse function or `migrations.RunPython.noop`.
2. **Never destructive on populated tables**: adding NOT NULL → provide a default or split into (a) add nullable, (b) backfill, (c) set NOT NULL.
3. **Every new FK gets an index**: `db_index=True` on the field or explicit `AddIndex`.
4. **Every new column used in WHERE/ORDER BY gets an index**.
5. **Data migrations are separate**: don't mix schema changes and data backfills in one migration.
6. **Don't edit applied migrations**: write a new one.
7. **Test forward and backward locally** before committing.
8. **Don't import models inside RunPython**: use `apps.get_model()`.

## Don't do

- Don't use `migrations.RunSQL` for anything `AlterField`/`AddField` covers.
- Don't disable FK checks.
- Don't drop a NOT NULL constraint on a column that's referenced by an index expression.
- Don't commit migrations without testing `migrate` and `migrate <app> <previous>`.
